Frequently Asked Questions

Everything you need to know about DevTrack's technology and permissions model.

DevTrack webhooks use a separate event-driven ingestion service. We bypass regular API rate limits by registering standard webhook endpoints directly, meaning we receive pushed payloads instead of polling. For heavy workflows, a Redis/BullMQ queue buffers updates to guarantee message delivery.

Yes. Workspace Owners can specify precise permissions (e.g. read, write, admin access) at both the workspace level and the individual project level. Our Role-Based Access Control (RBAC) ensures developers only view metrics for projects they are explicitly assigned to.

We take security extremely seriously. All repository metadata links use OAuth2 with restricted read-only permissions for commit information. We do not store or access your actual source code. All stored token data is encrypted using AES-256 at rest.

The Free Starter plan includes 1 developer seat and allows tracking of up to 3 active projects. To add more collaborators, expand access to detailed team performance charts, or run advanced Aria AI analyses, you can upgrade to our Pro plan.
